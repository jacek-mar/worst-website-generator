## ğŸ”¬ Assumptions & design notes

- ğŸ¯ **Deterministic RNG** â€” Uses Mulberry32 for reproducible results. Same seed
  always produces the same "bad" page. This is actually useful for debugging.
- ğŸ—‚ï¸ **Chaos settings are normalized** â€” All sliders clamp to 0-10 range, defaults
  are intentionally high (worst is the default).
- ğŸ“ **Landing page types share common structure** â€” Features, testimonials, FAQ,
  and form sections are generated similarly but with type-specific content.
- ğŸ‘¾ **Palette generation** â€” Neon colors are selected based on page type but
  intentionally clash. Backgrounds use CSS gradients that hurt the eyes.
- ğŸ¤” **Vibecoding has limits** â€” AI models are excellent at generating large
  amounts of coherent code quickly. They are less reliable at maintaining
  consistency across multiple files. Human review and targeted correction
